# Learning Tracks

Different goals require different paths through the repo.

## Track A - Conceptual researcher

Goal: understand the field and read papers confidently.

1. `curriculum/00-prerequisites.md`
2. `curriculum/01-foundations.md`
3. `notes/architecture-taxonomy.md`
4. `resources/papers.md`
5. `curriculum/03-training-pipeline.md`
6. `curriculum/05-alignment.md`
7. `notes/open-research-questions.md`

Minimum projects:

- Architecture comparison table
- Dataset matrix
- Evaluation taxonomy

## Track B - Builder / engineer

Goal: prototype and fine-tune practical VLM systems.

1. `curriculum/00-prerequisites.md`
2. `labs/01-clip-retrieval.md`
3. `labs/02-build-a-tiny-vlm.md`
4. `labs/03-finetune-small-vlm.md`
5. `labs/04-vlm-evaluation-harness.md`
6. `projects/project-ideas.md`

Minimum projects:

- CLIP retrieval demo
- Small VLM fine-tune or prompt-only baseline
- Evaluation harness

## Track C - Agent-focused multimodal systems

Goal: build systems that see screens/documents and act.

1. `curriculum/02-architectures.md`
2. `curriculum/06-evaluation.md`
3. `curriculum/07-audio-video-any-to-any.md`
4. `projects/project-ideas.md` Project 6 or 7
5. `notes/open-research-questions.md`

Minimum projects:

- Screenshot QA dataset
- Browser/screen action schema
- Verification loop

## Track D - Training and alignment specialist

Goal: understand data mixtures, SFT, DPO/RLHF, and regression risks.

1. `curriculum/03-training-pipeline.md`
2. `curriculum/04-datasets.md`
3. `curriculum/05-alignment.md`
4. `resources/datasets-benchmarks.md`
5. `labs/03-finetune-small-vlm.md`
6. `labs/04-vlm-evaluation-harness.md`

Minimum projects:

- Multimodal instruction dataset
- Preference dataset
- Before/after eval with text-only regression checks
